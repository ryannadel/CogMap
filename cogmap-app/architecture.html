How CogMap works

CogMap packages the data-processing engine inside an installable agent skill. The skill never treats its install folder as user data: it creates a local cogmap/ workspace, ingests notes from sources/, stores resumable intermediate artifacts in work/, and renders a portable visualization in output/.

Why this design makes sense

CogMap keeps deterministic code deterministic and lets the host agent do only the semantic steps that need judgment: concept extraction, entity resolution, and synthesis. Because every stage writes files, the loop is resumable after interruption and incremental after edits.

Important boundaries

These boundaries prevent the old confusion between pipeline code, scratch state, and final artifacts.

What is packaged

  • SKILL.md and the agent prompts.
  • Deterministic engine scripts in scripts/.
  • A bundled demo under assets/demo/ for first-run behavior.

What is user/project-local

  • The user's notes in cogmap/sources/.
  • LLM artifacts and refresh cache in cogmap/work/.
  • The rendered visualization in cogmap/output/.
